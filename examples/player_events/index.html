Registering and un-registering to player events

You can register and un-register to player events.
In order to establish communication, provide a unique "api" attribute and use it to invoke the player's "addEventListener" and "removeEventListener" methods.

Note: You must wait for the player ready event before registering to its events.
The directive dispatches the kalturaPlayerReady event and passes the api identifier as an event parameter.
Use this identifier to call the player's "addEventListener" and "removeEventListener" methods.

For a full list of available player events, look at the Kaltura API documentation.
